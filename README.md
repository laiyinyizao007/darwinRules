# Darwin Rules - 通用规则库

> AI 自我改进系统的通用规则集合

---

## 📚 简介

Darwin Rules 是一套为 AI 助手（如 Cline）设计的通用规则库，帮助 AI：
- 📝 记忆项目信息和决策
- 🎯 遵循最佳实践
- 🔄 持续改进工作流程

---

## 📁 目录结构

```
core/
├── rules.mdc                  # 基础规则
├── mcp-tools.mdc              # MCP 工具使用指南
├── darwin-guide.mdc           # Darwin 使用指南
├── directory-structure.mdc    # 目录结构规范
├── ACT/
│   └── implement.mdc          # ACT 模式实现流程
└── PLAN/
    └── plan.mdc               # PLAN 模式规划流程
```

---

## 🚀 如何使用

### 在项目中集成

1. 在项目根目录创建 `.clinerules` 目录
2. 使用同步脚本下载规则到本地
3. Cline 会自动读取并应用这些规则

详细步骤请参见各项目的 `.clinerules/README.md`

---

## 📖 规则说明

### 核心规则 (rules.mdc)
- 自动执行命令
- 优先安装而非替代
- MCP 工具优先使用
- 代码规范
- Git 提交规范

### 工作流程

**PLAN 模式** - 5 阶段规划流程：
1. 需求澄清
2. 探索方案
3. 方案评估
4. 详细设计
5. 用户确认

**ACT 模式** - 5 阶段实现流程：
1. 需求确认
2. 设计方案
3. 增量实现
4. 质量保证
5. 文档更新

---

## 🔄 更新规则

项目中使用 `dw-update` 命令同步最新规则：
- 智能比较本地和远程文件
- 保留最新版本
- 显示详细同步报告
